Install ROS

The ROS Wiki provides a complete installation guide. The following instructions are a summary of their article.

Attention

Since Ubuntu 18.04, Canonical decided it was a good idea to have unattended updates running in the background. This might lead to error messages similiar to:

E: Could not get lock /var/lib/dpkg/lock-frontend - open (11: Resource temporarily unavailable)
E: Unable to acquire the dpkg frontend lock (/var/lib/dpkg/lock-frontend), is another process using it?

If you encounter them after running anything related to apt: no worries. Just wait until the backgroud updates are finished.

Preparation

  1. Add sources

    sudo sh -c 'echo "deb http://packages.ros.org/ros/ubuntu $(lsb_release -sc) main" > /etc/apt/sources.list.d/ros-latest.list'

  2. Add key

    sudo apt install curl # if you haven't already installed curl
curl -s https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc | sudo apt-key add -

  3. Update index

    sudo apt update

  4. Make sure your package are up to date!(!!!)

    sudo apt upgrade -y

Installation

  1. Install ROS

    sudo apt install ros-noetic-desktop-full

  2. Install build dependencies

    sudo apt install python3-rosdep python3-rosinstall python3-rosinstall-generator python3-wstool build-essential

  3. Install catkin-tools

    sudo apt install python3-catkin-tools python3-osrf-pycommon

  4. Set up rosdep

    sudo rosdep init

    rosdep update

Set Up Catkin Workspace

  1. Source your ROS installation

    source /opt/ros/noetic/setup.bash

  2. Create workspace

    mkdir -p ~/fav/catkin_ws/src && cd ~/fav/catkin_ws

  3. Initialize workspace

    catkin init

  4. Build empty workspace

    catkin build

To automatically source your catkin workspace, execute the following command:

echo 'source /opt/ros/noetic/setup.bash' >> ~/.bashrc
echo 'source $HOME/fav/catkin_ws/devel/setup.bash' >> ~/.bashrc

To apply these changes, execute:

source ~/.bashrc

For new terminal session your catkin workspace is sourced automatically since we added the instruction to do so to the .bashrc file.

Attention

If you are using zsh instead of bash, you need to adjust the above commands. You will also have pay attention and adjust some commands in the following parts of this setup guide.

Get The First Catkin Packages

  1. Make sure you are inside your catkin workspace:

    cd ~/fav/catkin_ws

  2. Clone bluerov_sim

    git clone https://github.com/FormulasAndVehicles/fav.git ~/fav/catkin_ws/src/fav

    At this stage make sure your directory structure looks similar to this:

    /home/YOUR_USER_NAME/fav/catkin_ws
├── build
├── devel
├── logs
└── src
    └── fav
        ├── fav_gazebo_plugins
        ├── fav_msgs
        └── fav_sim

  3. Let ROS resolve the package’s dependencies

    rosdep install --from-paths src --ignore-src -r -y

  4. Rebuild your workspace

    catkin build

  5. Apply changes of your environment variables by either

    • starting a new terminal session (remember that this means ~/.bashrc gets sourced automatically as mentioned before).

    • sourcing ~/.bashrc manually by executing

      source ~/.bashrc

Test the Setup

Hint

If launching gazebo from ROS, ROS is not able to kill the gazebo node without escalating to SIGTERM. This happens after a 15s timeout. If you do not have the time to wait for so long, you can modifiy /opt/ros/noetic/lib/python3/dist-packages/roslaunch/nodeprocess.py and change DEFAULT_TIMEOUT_SIGINT to some value you are willing to wait.

If you are willing to wait, let’s say, 3.0 seconds, execute the following command:

sudo sed -i 's/DEFAULT_TIMEOUT_SIGINT\s*=\s*15\.0.*/DEFAULT_TIMEOUT_SIGINT = 3\.0/' /opt/ros/noetic/lib/python3/dist-packages/roslaunch/nodeprocess.py

Execute the following command

roslaunch fav_sim gazebo_apriltag_tank_world.launch

and gazebo should launch after a few seconds and look this:

If the gazebo window stays black, enter Ctrl + C in the terminal in which you executed the command above and try it again.
