ROS Resources

Note

This section tries to give you an overview of additional resources and tools you might need when working with ROS. This is by no means complete, and we encourage you to explore what’s out there on your own.

You can find the ROS wiki here.

ROS Tutorials

The official ROS tutorials are a very good place to start if you’ve never used ROS before and got some time on your hands. There are C++ and Python versions for coding examples. Since we will be using Python in this class, you can skip C++ specific chapters. However, if you’re already confident in C++ (more so than in Python), feel free to use C++ instead.

Note

There are a lot of other (beginner’s) tutorials for ROS online. If the official ROS tutorials aren’t for you, just have a look on google.

Commandline Tools

There are a lot of commandline tools. Apart from roslaunch, tools that you might find helpful include rostopic, rosbag, rosparam and roscd. When encountering problems with your ROS setup, roswtf can be very useful.

Visualization Tools

Once you start working with ROS and are writing your own nodes, you will want to be able to visualize data and analyze interaction between nodes.

rqt

rqt consists of various GUI tools, almost all of them are useful. We make use of the Topic Monitor and rqt Multiplot the most. For testing, the Message Publisher is also very useful.

  1. To install Multiplot, run

    sudo apt install ros-noetic-rqt-multiplot

  2. To tell rqt about this new plugin, run

    rqt --force-discover

Now, you will be able to use the Multiplot plugin from the rqt GUI.

RViz

RViz is an awesome tool for 3D visualization, but can take some time getting used to it. RViz can visualize different robot coordinate frames, for coordinate transformations in ROS, look here: Coordinate Transformations

Recording and playing back data

ROSbags are an awesome functionality of ROS, trust me. Use them. There also is a ROS tutorial on how to record and play back data.

In case you want to then plot your data in, for example, Matlab, converting your ROSbags to *.csv files will be useful. I like this tool for that purpose, but there are other options out there.

We now also have a tutorial available in the section Recording Data Using Bag Files.

Coordinate Transformations

ROS uses the tf transformations tool to keep track of coordinate frames and their relationships over time. The second and newest generation of the transform library is tf2 .

If you want to transform vectors by hand using rotation matrices or quaternions, you might find the transformations (Python) API helpful.

The tf.transformations library also helps you deal with quaternions by providing various transformation options, for example to Euler angles.

Messages

There’s a pre-defined ROS message for almost everything, trust me. Standard messages representing primitive data types and other basic message constructs, such as multiarrays, can be found in std_msgs. The package common_msgs includes the most common generic robot-specific message packages, such as geometry_msgs, sensor_msgs and nav_msgs. When you google a message name, you’ll easily find the API (example for a sensor_msgs/Imu.msg ).

However, if you do find yourself needing to define your own message type, there is a ROS tutorial on that, too.

ROS Logger

Of course you can use simple print() commands. However, ROS has a logger that you can (should) use for printing messages to your console if they are not for debugging purposes only. There is, of course, a tutorial.

Installing additional Packages

Released packages can be installed with aptitude and don’t have to be build from source. Use:

sudo apt install ros-noetic-<package>
